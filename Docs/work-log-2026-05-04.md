# 📓 프로젝트 작업 로그 (2026-05-04)

## 📋 작업 요약
- **목표**: 정적 프로젝트 관리 시스템을 Google Sheets(GAS) 연동 기반의 동적 시스템으로 전환 및 UI 최적화.
- **상태**: 구현 완료 및 검증 성공.

## 🛠️ 주요 구현 내용
1. **Google Apps Script (GAS) 연동**:
   - `doGet` (조회) 및 `doPost` (추가) 기능을 포함한 백엔드 API 구축.
   - 구글 스프레드시트를 데이터베이스로 활용하여 데이터 영속성 확보.
2. **동적 프로젝트 대시보드**:
   - `useState` 및 `useEffect`를 통한 실시간 데이터 페칭 구현.
   - 데이터 로딩 시 스켈레톤 스타일의 로딩 애니메이션 적용.
3. **프리미엄 프로젝트 추가 모달**:
   - 글래스모피즘 디자인이 적용된 입출력 폼 구현.
   - 아이콘, 강조 색상, 카테고리를 드롭다운 메뉴(Select)로 선택하도록 개선하여 편의성 및 데이터 정확도 향상.

## ⚠️ 발생했던 문제 및 해결 방법 (Troubleshooting)
1. **데이터 미표시 문제 (Hydration & Animation)**:
   - **원인**: 서버/클라이언트 하이드레이션 불일치 및 Framer Motion의 애니메이션 상태 고착.
   - **해결**: `isMounted` 상태를 통한 클라이언트 전용 렌더링 적용 및 `isReady` 상태를 이용한 애니메이션 강제 트리거(`animate` 속성 사용).
2. **Key 매핑 및 대소문자 문제**:
   - **원인**: 구글 시트의 헤더명과 코드상의 변수명(CamelCase) 불일치.
   - **해결**: 데이터 페칭 단계에서 모든 키를 소문자로 변환한 뒤, 필요한 키(`accentColor` 등)만 CamelCase로 다시 매핑하는 정규화 로직 도입.
3. **보안 이슈**:
   - **원인**: GAS URL 하드코딩.
   - **해결**: `.env.local` 환경 변수로 분리하고 `.gitignore` 설정을 통해 소스 코드 노출 방지.
