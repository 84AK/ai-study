# 작업 로그: 수업 자료 관리 시스템 (Admin Material System) 구축

**날짜:** 2026-04-28
**담당:** 건축가(Architect), 작업자(Worker), 디자이너(Designer), 서기(Scribe)

## 1. 요구사항 분석
- 사용자가 직접 앱 화면에서 수업 자료(노션 URL 등)를 추가/수정/삭제하기를 원함.
- 특정 계정(ID: `mosimose`, PW: `Ahtpdi12qw?`)으로 로그인했을 때만 관리 가능해야 함.
- 외부 DB(Supabase 등) 없이 로컬 환경에서 효율적으로 작동해야 함.

## 2. 구현 내용
### 2.1 데이터 아키텍처 (JSON + API)
- **저장소**: `src/data/materials.json` 파일 생성.
- **백엔드**: Next.js API Route(`src/app/api/materials/route.ts`)를 구축하여 파일 시스템(`fs`) 기반의 읽기/쓰기/삭제 기능 구현.
- **보안**: API 레벨에서 비밀번호 검증 로직 추가.

### 2.2 어드민 대시보드 (`/admin`)
- **UI/UX**: Glassmorphism 및 Bento Grid 스타일의 세련된 관리자 화면 설계.
- **기능**:
    - 로그인 폼 (보안 접속).
    - 수업 자료 추가 폼 (제목, URL, 카테고리).
    - 등록된 자료 목록 조회 및 개별 삭제 기능.
    - `framer-motion`을 이용한 부드러운 전환 효과.

### 2.3 메인 페이지 연동 (`/`)
- **데이터 바인딩**: `useEffect`를 통해 API로부터 실시간 자료 목록을 가져오도록 수정.
- **수업 자료 섹션**: 벤토 그리드 레이아웃에 '수업 자료 보관함' 섹션 추가.
- **반응형 디자인**: 모바일 및 데스크탑 최적화.

## 3. 결과 및 향후 계획
- 이제 사용자님은 `/admin` 페이지에서 편리하게 수업 자료를 관리할 수 있습니다.
- 로컬 환경(`npm run dev`)에서 즉시 반영되며, 깃허브 업로드 시 전체 사이트에 공유됩니다.
- 추후 배포 환경에 따라 Supabase나 Google Sheets 연동으로 확장 가능합니다.
